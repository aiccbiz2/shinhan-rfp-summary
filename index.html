신한카드 AI 기반 상담품질 관리
및 Assistant 시스템 구축

콜센터 녹취를 STT로 변환하고 생성형 AI로 자동 평가하는 AI-QA 통화품질평가 시스템과 STT 분석 기능을 신규 구축하는 사업이다. 다부서(컬렉션·CRM·심사발급·준법지원·소비자보호)가 요구 주체이며, 평가 결과는 사내 제우스 시스템과 연동된다. AI상담 Assistant는 PDS 고도화 완료('27.4月) 후 도입 예정으로 범위·일정 불확실성이 최대 리스크다.

최대 리스크: 범위/일정 불확실 AI-QA 통화품질평가 STT 분석 AI상담 Assistant 제우스 연동
01

사업 개요

신한카드가 발주한 콜센터/컨택센터 대상 AI 기반 상담품질 관리 시스템 신규 구축 사업이다. 핵심은 (1) 통화 녹취를 STT로 변환해 생성형 AI/LLM으로 자동 평가하는 AI-QA 통화품질평가 시스템, (2) STT 데이터의 분류·금칙어 탐지·검색·KMS 관리 기능, (3) 실시간 상담을 지원하는 AI상담 Assistant(본 입찰에서는 제언 평가) 도입이다. 다수의 현업 부서가 요구 주체로 참여하며, 평가 결과는 사내 인사·성과 시스템(제우스)과 연동되어 월별 종합평가에 반영된다.

02

사업 범위 구성

구분내용도입 시점
1. AI-QA 통화품질평가 시스템어플리케이션 개발 (부서별 요구사항)본 사업
2. STT 분석 기능소비자보호부 주관 — STT 평가·분류·금칙어·플레이어·AI 채팅·KMS본 사업
3. AI상담 Assistant실시간 상담지원. 본 입찰은 제언을 받아 평가만 진행PDS 고도화 완료('27.4月) 및 기술 검증 後
03

요구사항 상세

AI-QA 통화품질평가 — 컬렉션관리부통화품질평가 / 상담원·채권팀장·본사평가 UI

통화품질평가(공통)

  • 평가 항목별 필수 안내 여부 확인·반영
  • 권한 스크립트·필수 안내·금칙어 평가(연체경과일수·결제금액·거래정지·가용한도 등)
  • 불친절 표현·언쟁·말 끊기 등 응대 행위 탐지·평가
  • 고객 요청 처리 여부 확인(결제방법·결제일 변경, SMS발송 등)
  • 음성사서함 녹음통화 집계·평가
  • 추심 횟수 제한 준수 확인(1일 2회 / 7일 이내 7회)
  • 실질 통화연결콜 평가, 통화대상별 포함 여부 판단

상담원 / 채권팀장 / 본사평가 UI

  • 평가콜 조회(일/주/월, 평균·항목별 평균), 일별 콜 평가 레포트(전일·전월 비교)
  • 사후처리 통화 자동 분류·처리 확인, 우수콜 등록·조회·청취
  • 키워드 검색(금감원·결제일변경·환불 등), 이의제기 등록·수정·확정
  • 콜 연습 기능, 평가 기준·배점 등록/관리, 기간별 제외·감안 기준 관리
AI-QA 통화품질평가 — CRM부마케팅(판촉) 유형별 평가
  • 마케팅 유형별 평가 항목 분류 관리(항목 등록·수정·삭제, 비중 관리) — 제우스 판촉코드/판촉명 연동
  • 공통 필수(첫인사·본인확인·소속안내·종료인사) + 규제 필수(금소법 설명의무·적합성, 모범규준, 취약계층 재안내) 평가
  • 척도: Y/N 또는 준수율(%) 모두 가능하도록 개발
  • 콜품질(응대스킬) 차감 평가 — TM 속도·발음, 정성 응대 등
  • 금칙어·비하어 사용콜 목록·건수, STT 조건·키워드·문장 검색 및 녹취 연동
  • 종합평가 의견 + 오안내 정제 안내 문장 제안
  • 협력사별·직무별·상담사별 점수화, 상위·하위 5~10% 콜 목록
  • 기존 시스템 연동 점검(두낫콜·카드발급·월납·SMS·7609BL·주소/계좌) — 연동방법 논의 필요
  • 월단위 종합점수 제우스 연동 필수(월별 모니터링 반영 중)
AI-QA 통화품질평가 — 심사발급부 / 준법지원부발급·심사 / 준법 영역

심사발급부

  • 권한 스크립트·필수 안내·금칙어 평가(본인확인·신청여부·카드수령지·배송 등)
  • 고객 요청 처리 확인(결제일·결제계좌·단기카드대출 등)
  • 당사 시스템 기준 확인 필요: 채널(온라인/설계사/은행)·카드(신용/체크)·나이(성인/미성년)·내외국인·발급기준(신규/재발급/가족카드)
  • 심사자·S팀장·본사평가 UI(조회·레포트·사후처리·이의제기·키워드 검색)
  • 월납 등록 필수안내(아파트관리비·통신비), 마케팅 유무 확인(퍼미션/단기카드대출)

  • 평가 기준·배점 등록/관리, 키워드 언급 현황 분석·리포트
소비자보호부 — STT 분석 기능평가·분류·금칙어·플레이어·AI 채팅·KMS

STT 평가·분류

  • STT·평가요약·상담요약·세부내용 유기적 조회 + 평가 확인용 AI 채팅
  • 대/중/소분류, 기존 값 없으면 AI 분류유형 자동 제안 → 담당자 컨펌 확정
  • 반복민원·고객제안 등 소분류 리포팅·알림

금칙어 · 플레이어 · AI 채팅 · KMS

04

핵심 사실 추출

요구사항·기능

거의 모든 항목이 "필수". STT 평가 일부(프롬프트 평가기준 관리)만 "선택". 생성형 AI 의존 기능 다수(자동 평가의견·분류 제안·금칙어 오타 제안·스크립트 자동 생성).

SLA·서비스 수준

정량 SLA(가동률·복구시간) 미명시. 단, 실시간 STT 출력·KMS 검색 소요시간을 "타사 대비 경쟁력 있는 시간"으로 제시 요구 → 사실상 성능 약속.

일정·납기

명시 일자는 PDS 고도화 완료 '27.4月뿐. 본 사업(AI-QA·STT) 착수·납기·마일스톤은 문서 미명시.

평가기준·배점

입찰 기술/가격 배점·자격 요건 미명시. 문서 내 "평가 기준·배점"은 시스템이 다룰 상담품질 평가 항목을 의미.

계약·법적 조건

페널티·지체상금·하자보수 미명시. 규제 준수 요구는 존재: 금소법, 모범규준, 취약계층 재안내, 추심 횟수 제한(1일 2회/7일 7회).

인력·체계

투입 인력 자격·조직 요구 미명시. 권한 체계는 명확: 상담원/팀장(S팀장)/본사평가/운영관리자 + 협력사 구조.

솔루션·아키텍처

연동 대상: 제우스·GEUS·계정계 I/F·DW(일배치)·SSO. STT 서버, 독립 KMS 신규 환경, 학습·재학습 파이프라인.

특이·주의 조항

"연동방법 논의 필요", "당사 시스템 확인 필요"(비고 공란) 다수. AI상담 Assistant는 PDS·기술검증 종속 조건부 사업.

05

관점별 분석

제안서 작성 담당자
  • 전 항목 필수 + 평가배점·일정·예산 미명시 → 제언 품질·기술검증 능력이 수주 차별화 요소.
  • 실시간 STT·KMS 검색 소요시간 정량 제시 요구 = SLA 약속 직결 → 근거·여유 확보 후 제시.
  • 척도가 Y/N·준수율(%) 모두 지원 → 평가 엔진 유연성·노코드 운영을 강점화.
  • 금칙어·분류 자동 제안 정확도가 평가 대상 가능 → 검증 데이터·정확도 지표 포함.
고객사 영업 담당자
  • 요구 주체가 5개 부서로 분산·상이 → 이해관계자·우선순위 조정이 핵심.
  • AI상담 Assistant가 PDS 종속 → 제언 vs 실제 구축 단계 분리 합의로 분쟁 예방.
  • 제우스·계정계 의존도 높음 → 고객 IT 부서 협조·일정 확보가 성패 좌우.
  • 협력사 성과 평가(상위·하위 5~10% 노출)의 민감성 사전 조율 필요.
프로젝트 매니저
  • 일정·예산·평가배점·SLA·페널티가 문서에 없음 → 리스크 산정 전 자료 즉시 요청.
  • "연동방법 논의 필요"·비고 공란 등 미확정 다수 → Scope Creep 리스크 관리.
  • AI상담 Assistant '27.4月 PDS 종속을 본 사업과 분리, 책임 범위 계약 명확화.
  • 다층 사용자·권한(상담원/팀장/본사/운영/협력사)으로 UI·권한 설계 공수 큼.
프로젝트 개발자
  • 실시간 STT(화자 분리·저지연), 오타 금칙어, AI 분류·평가·답변서 봇 등 생성형 AI 다수 → 정확도 검증 부담.
  • 제우스·GEUS·계정계 I/F·DW·SSO 등 연동 인터페이스 다수, 통화 중 정보 자동 연동.
  • 독립 KMS 신규 + 원본 자동 표준화/변환, 스크립트 자동 생성·재학습 파이프라인 설계.
  • 평가 미완료 콜 STT 생성, STT-녹취 위치 동기화(클릭 재생) 등 실시간·동기화 난이도.
